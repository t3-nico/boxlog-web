# Building a Blog with Next.js and Tailwind CSS

This article provides a comprehensive guide on building a modern blog site using Next.js and Tailwind CSS with detailed examples and best practices.

## Introduction

In modern web development, **performance** and **development efficiency** are crucial elements. Next.js is a React framework with the following key features:

- Server-Side Rendering (SSR)
- Static Site Generation (SSG)
- Automatic Code Splitting
- Image Optimization

## Setup

First, create a new Next.js project:

```bash
npx create-next-app@latest my-blog
cd my-blog
npm install
```

Next, add Tailwind CSS:

```bash
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p
```

## Blog Structure

A blog site requires the following structure:

1. **Article List Page** - Display all articles
2. **Individual Article Page** - Show article details
3. **Tag Pages** - Article list by tags
4. **Search Functionality** - Search articles

### Introducing MDX

We use MDX for writing articles. MDX offers the following advantages:

- Concise Markdown syntax
- Use React components within Markdown
- Create dynamic content

```javascript
import { MDXRemote } from 'next-mdx-remote/rsc'

const components = {
  h1: (props) => <h1 className="text-3xl font-bold" {...props} />,
  p: (props) => <p className="text-gray-700 leading-relaxed" {...props} />,
}

export default function BlogPost({ content }) {
  return (
    <div className="prose prose-lg max-w-none">
      <MDXRemote source={content} components={components} />
    </div>
  )
}
```

## Styling

Using Tailwind CSS provides the following benefits:

- **Consistent Design** - Styling based on design systems
- **Rapid Development** - Quick styling with preset classes
- **Responsive Design** - Mobile-first approach

### Card Component

Sample article card:

```jsx
function PostCard({ post }) {
  return (
    <article className="bg-white rounded-lg shadow-md overflow-hidden hover:shadow-lg transition-shadow">
      <img 
        src={post.coverImage} 
        alt={post.title}
        className="w-full h-48 object-cover"
      />
      <div className="p-6">
        <h2 className="text-xl font-bold mb-2">{post.title}</h2>
        <p className="text-gray-600 mb-4">{post.description}</p>
        <div className="flex items-center justify-between">
          <span className="text-sm text-gray-500">{post.publishedAt}</span>
          <div className="flex gap-2">
            {post.tags.map(tag => (
              <span key={tag} className="px-2 py-1 bg-blue-100 text-blue-800 text-xs rounded">
                #{tag}
              </span>
            ))}
          </div>
        </div>
      </div>
    </article>
  )
}
```

## Performance Optimization

### Image Optimization

Using Next.js's `next/image` component automatically optimizes images:

```jsx
import Image from 'next/image'

function CoverImage({ src, alt }) {
  return (
    <Image
      src={src}
      alt={alt}
      width={1200}
      height={630}
      className="w-full h-auto rounded-lg"
      priority
    />
  )
}
```

### Static Generation

Blog articles are pre-generated using `generateStaticParams`:

```javascript
export async function generateStaticParams() {
  const posts = await getAllPosts()
  
  return posts.map((post) => ({
    slug: post.slug,
  }))
}
```

## SEO Optimization

Properly setting metadata can enhance SEO effectiveness:

```javascript
export async function generateMetadata({ params }) {
  const post = await getPost(params.slug)
  
  return {
    title: post.title,
    description: post.description,
    openGraph: {
      title: post.title,
      description: post.description,
      images: [post.coverImage],
    },
  }
}
```

## Conclusion

Combining Next.js and Tailwind CSS provides the following benefits:

- **Fast Performance** - Quick loading with SSG
- **Excellent Developer Experience** - Hot Reload and type safety
- **Maintainable Code** - Component-based design
- **SEO-Friendly** - Proper metadata and SSR

Using these technologies, you can build modern, high-performance blog sites.
