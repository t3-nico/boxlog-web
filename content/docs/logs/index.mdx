## Quick Reference

### Log Levels Quick Guide
```javascript
logger.trace('Detailed execution flow')    // Development only
logger.debug('Debugging information')      // Development/staging
logger.info('Important business events')   // All environments
logger.warn('Potentially harmful')         // All environments
logger.error('Error conditions')           // All environments
logger.fatal('Critical system errors')     // All environments
```

### Structured Logging Template
```javascript
logger.info('Event description', {
  event: 'user_action',
  userId: 'user_123',
  action: 'purchase',
  amount: 99.99,
  timestamp: new Date().toISOString(),
  metadata: {
    source: 'web',
    version: '1.2.3'
  }
})
```

## Best Practices Summary

### ‚úÖ Do
- Use structured logging with consistent schemas
- Include relevant context in every log
- Use appropriate log levels
- Implement correlation IDs for distributed systems
- Log both successes and failures
- Include performance metrics

### ‚ùå Don't
- Log sensitive information (passwords, tokens, PII)
- Use only string concatenation for messages
- Log everything at the same level
- Ignore error stack traces
- Log without context
- Create log noise with excessive debugging
