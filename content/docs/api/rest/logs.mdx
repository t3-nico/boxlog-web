# Logs API

The Logs API is the core of BoxLog, allowing you to send, search, and manage log data programmatically.

## Base URL

```
https://api.boxlog.io/v1
```

## Authentication

All API requests require authentication using your API key:

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" \
  https://api.boxlog.io/v1/logs
```

## Endpoints

### Send Logs

Send one or more log entries to BoxLog.

```http
POST /logs
```

#### Request Body

```json
{
  "logs": [
    {
      "timestamp": "2025-01-16T10:30:00Z",
      "level": "info",
      "message": "User login successful",
      "service": "auth-service",
      "metadata": {
        "userId": "user_123",
        "ip": "192.168.1.1",
        "userAgent": "Mozilla/5.0..."
      }
    }
  ]
}
```

#### Response

```json
{
  "success": true,
  "accepted": 1,
  "errors": []
}
```

#### Example

```javascript
const response = await fetch('https://api.boxlog.io/v1/logs', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    logs: [{
      timestamp: new Date().toISOString(),
      level: 'info',
      message: 'Payment processed',
      service: 'payment-service',
      metadata: {
        amount: 99.99,
        currency: 'USD',
        customerId: 'cust_123'
      }
    }]
  })
})
```

### Search Logs

Search and filter logs with powerful query capabilities.

```http
GET /logs/search
```

#### Query Parameters

| Parameter | Type | Description | Example |
|-----------|------|-------------|---------|
| `q` | string | Search query | `error AND service:api` |
| `from` | string | Start time (ISO 8601) | `2025-01-16T00:00:00Z` |
| `to` | string | End time (ISO 8601) | `2025-01-16T23:59:59Z` |
| `level` | string | Filter by log level | `error,warning` |
| `service` | string | Filter by service | `auth-service` |
| `limit` | integer | Results per page (max 1000) | `100` |
| `offset` | integer | Pagination offset | `0` |

#### Response

```json
{
  "logs": [
    {
      "id": "log_abc123",
      "timestamp": "2025-01-16T10:30:00Z",
      "level": "error",
      "message": "Database connection failed",
      "service": "api",
      "metadata": {
        "error": "ECONNREFUSED",
        "host": "db.example.com",
        "port": 5432
      }
    }
  ],
  "total": 42,
  "page": {
    "limit": 100,
    "offset": 0
  }
}
```

### Get Log by ID

Retrieve a specific log entry by its ID.

```http
GET /logs/{log_id}
```

#### Response

```json
{
  "id": "log_abc123",
  "timestamp": "2025-01-16T10:30:00Z",
  "level": "error",
  "message": "Database connection failed",
  "service": "api",
  "metadata": {
    "error": "ECONNREFUSED",
    "host": "db.example.com",
    "port": 5432
  },
  "context": {
    "traceId": "trace_123",
    "spanId": "span_456"
  }
}
```

### Delete Logs

Delete logs matching specific criteria.

```http
DELETE /logs
```

#### Request Body

```json
{
  "query": "service:test-service",
  "from": "2025-01-01T00:00:00Z",
  "to": "2025-01-15T23:59:59Z"
}
```

#### Response

```json
{
  "deleted": 1523,
  "duration": "2.3s"
}
```

## Log Levels

BoxLog supports the following standard log levels:

| Level | Value | Description |
|-------|-------|-------------|
| `trace` | 10 | Detailed trace information |
| `debug` | 20 | Debug messages |
| `info` | 30 | Informational messages |
| `warn` | 40 | Warning messages |
| `error` | 50 | Error messages |
| `fatal` | 60 | Fatal errors |

## Query Syntax

BoxLog uses a powerful query language for searching logs:

### Basic Queries
- `error` - Find logs containing "error"
- `"database connection"` - Exact phrase match
- `user_id:123` - Field matching

### Boolean Operators
- `error AND database` - Both terms must match
- `error OR warning` - Either term matches
- `error NOT test` - Exclude logs with "test"

### Field Queries
- `level:error` - Match specific log level
- `service:api` - Match specific service
- `metadata.userId:123` - Nested field matching

### Range Queries
- `response_time:>100` - Greater than 100
- `status:[400 TO 499]` - Range inclusive
- `timestamp:[2025-01-16 TO 2025-01-17]` - Date range
