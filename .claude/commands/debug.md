# デバッグ

問題を調査し、原因を特定して修正してください。

## デバッグワークフロー

### 1. 問題の理解

- エラーメッセージを確認
- 再現手順を把握
- 期待される動作を確認

### 2. 原因の調査

- 関連コードを検索
- エラーのスタックトレースを分析
- 最近の変更を確認（git log）

### 3. 仮説の検証

- console.log / デバッガで確認
- 単体テストで動作確認
- 段階的に原因を絞り込み

### 4. 修正と検証

- 最小限の修正で対応
- 型チェック実行
- 既存テストが通ることを確認

## Think ツール活用

複雑な問題の場合：

- `think` - 簡単な問題
- `think hard` - 複雑なロジック
- `ultrathink` - アーキテクチャに関わる問題

[動作確認の手順]
