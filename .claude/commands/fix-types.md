# 型エラー修正

`npm run type-check` を実行し、すべての型エラーを修正してください。

## ワークフロー

1. `npm run type-check` を実行
2. エラーを分析
3. 各エラーを修正（any型は使用禁止）
4. 再度 `npm run type-check` を実行して確認
5. 全エラー解消まで繰り返す

## 修正時の注意

- `any` 型は絶対に使用しない
- `unknown` も避け、具体的な型を定義する
- 型推論が効く場合は明示的な型注釈は不要
- Zodスキーマから型を推論する場合は `z.infer<typeof schema>` を使用

## 出力形式

修正が完了したら、以下の形式で報告：

```markdown
## 型エラー修正完了

### 修正したファイル

- [ファイルパス]: [修正内容の概要]

### 型チェック結果

[npm run type-checkの出力]
```
